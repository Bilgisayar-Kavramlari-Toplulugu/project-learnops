# **Kodun Kilitli Kasası: CI/CD Secrets ve Güvenlik**

Bir CI/CD pipeline'ı, uygulamanızı sunucuya taşırken veya test ederken hassas bilgilere ihtiyaç duyar. Örneğin; veri tabanına bağlanmak için bir şifreye veya bildirim göndermek için bir Firebase anahtarına ihtiyacı vardır. Ancak bu bilgiler asla .yml dosyasının içine açık metin olarak yazılmamalıdır.

## **1\. Secret (Sır) Nedir?**

**Secret**, GitHub veya GitLab gibi platformların sunduğu, yalnızca pipeline çalışırken erişilebilen şifrelenmiş değişkenlerdir.

* Siz bir "Secret" oluşturduğunuzda (örneğin: DB\_PASSWORD), değerini girersiniz ve "Kaydet" dediğiniz an GitHub bunu şifreler.
* Kaydettikten sonra siz bile o şifreyi bir daha göremezsiniz; sadece pipeline (Runner) o değeri okuyabilir.

## **2\. Secrets vs. Environment Variables (Çevre Değişkenleri)**

Her değişken "sır" olmak zorunda değildir. Aradaki farkı şu tabloyla özetleyebiliriz:

| Özellik | Environment Variables (Vars) | Secrets |
| :---- | :---- | :---- |
| **İçerik** | API URL'si, Port numarası, Uygulama adı. | API Key, DB Şifresi, SSH Anahtarı. |
| **Görünürlük** | Herkes görebilir, .yml içinde durabilir. | Şifrelenmiştir, kimse göremez. |
| **Güvenlik** | Düşük. | Çok Yüksek. |

## **3\. Pipeline İçinde Kullanım**

Bir YAML dosyasında bu sırra erişmek için özel bir sözdizimi kullanılır. GitHub Actions'ta bu kullanım şu şekildedir:
```bash
- name: Uygulamayı Dağıt
  env:
    DB_PASS: ${{ secrets.DATABASE\_PASSWORD }}
  run: python deploy.py
```
## **4\. Masking (Maskeleme) Güvenliği**

GitHub Actions oldukça akıllıdır. Eğer pipeline loglarında (çıktılarında) yanlışlıkla bir Secret'ı ekrana yazdırırsanız (echo $SECRET), GitHub bunu otomatik olarak algılar ve loglarda şifre yerine **\*\*\*** gösterir.

Bu özellik, yanlışlıkla sızmaları ve log dosyalarının ele geçirilmesi durumunda oluşabilecek riskleri önleyen harika bir güvenlik katmanıdır.

### **Güvenli Yönetim İçin Altın Kurallar**

1. **Kod İçine Gömme (Hardcoding):** Asla şifreleri doğrudan kod dosyalarına yazmayın.
2. **En Az Yetki Prensibi:** Bir API anahtarı oluştururken ona sadece ilgili pipeline'ın ihtiyacı olan yetkileri verin.
3. **Düzenli Güncelleme:** Kritik şifreleri ve anahtarları belirli periyotlarla yenileyin.
