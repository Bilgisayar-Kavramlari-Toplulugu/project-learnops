# GCP Provider ile Cloud Kaynağı

Terraform'un teorik temelleri hazır — şimdi gerçek cloud kaynakları oluşturuyoruz. Bu bölümde Google Cloud Platform (GCP) provider'ı ile kimlik doğrulamadan Cloud Run, Cloud SQL ve GCS bucket yönetimine kadar kapsamlı bir uygulama yapıyoruz.

---

## GCP Provider'ı Tanıma

GCP provider, Terraform'un Google Cloud API'leriyle konuşmasını sağlayan plugin'dir. HashiCorp ve Google ortak olarak geliştirmektedir.

```hcl
terraform {
  required_version = ">= 1.7.0"

  required_providers {
    google = {
      source  = "hashicorp/google"
      version = "~> 5.0"
    }
    google-beta = {
      source  = "hashicorp/google-beta"
      version = "~> 5.0"
    }
  }
}

provider "google" {
  project = var.proje_id
  region  = var.bölge
  zone    = "${var.bölge}-a"
}
```

**`google` vs `google-beta`:**
- `google`: Stabil, GA (Generally Available) özellikler
- `google-beta`: Beta aşamasındaki yeni özellikler (dikkatli kullanın)

## GCP Kimlik Doğrulama

Terraform'un GCP'ye erişmesi için kimlik doğrulama gerekir. Üç yöntem vardır:

### 1. Application Default Credentials (Geliştirme İçin)

```bash
# gcloud CLI kurulumu sonrası
gcloud auth application-default login

# Hangi hesabın kullanıldığını kontrol et
gcloud auth list
gcloud config get-value project
```

Terraform, `GOOGLE_APPLICATION_CREDENTIALS` ortam değişkeni veya ADC'yi otomatik olarak bulur.

### 2. Service Account Key (CI/CD İçin)

```bash
# Service account oluştur
gcloud iam service-accounts create terraform-sa \
  --display-name="Terraform Service Account"

# Gerekli rolleri ver
gcloud projects add-iam-policy-binding PROJE_ID \
  --member="serviceAccount:terraform-sa@PROJE_ID.iam.gserviceaccount.com" \
  --role="roles/editor"

# Key dosyası oluştur
gcloud iam service-accounts keys create terraform-key.json \
  --iam-account=terraform-sa@PROJE_ID.iam.gserviceaccount.com
```

```hcl
provider "google" {
  credentials = file("terraform-key.json")  # veya env variable
  project     = var.proje_id
  region      = var.bölge
}
```

> ⚠️ **Güvenlik:** `terraform-key.json` dosyasını asla Git'e commit etmeyin! `.gitignore`'a ekleyin.

### 3. Workload Identity Federation (GitHub Actions İçin — En Güvenli)

```yaml
# .github/workflows/terraform.yml
- name: Authenticate to GCP
  uses: google-github-actions/auth@v2
  with:
    workload_identity_provider: ${{ secrets.GCP_WIF_PROVIDER }}
    service_account: ${{ secrets.GCP_SA_EMAIL }}
```

```hcl
# provider.tf — GitHub Actions'da key dosyası gerekmez
provider "google" {
  project = var.proje_id
  region  = var.bölge
  # Kimlik bilgileri ortam değişkenlerinden otomatik alınır
}
```

## Temel GCP Kaynakları

### 1. VPC Network

```hcl
# Network
resource "google_compute_network" "main" {
  name                    = "${var.proje_adi}-vpc"
  auto_create_subnetworks = false  # Manuel subnet yönetimi
  routing_mode            = "REGIONAL"
}

# Subnet
resource "google_compute_subnetwork" "backend" {
  name          = "backend-subnet"
  ip_cidr_range = "10.0.0.0/24"
  region        = var.bölge
  network       = google_compute_network.main.id

  private_ip_google_access = true  # Cloud SQL için gerekli
}

# Firewall — HTTP/HTTPS izni
resource "google_compute_firewall" "allow_web" {
  name    = "${var.proje_adi}-allow-web"
  network = google_compute_network.main.name

  allow {
    protocol = "tcp"
    ports    = ["80", "443"]
  }

  source_ranges = ["0.0.0.0/0"]
  target_tags   = ["web-server"]
}
```

### 2. Cloud Storage (GCS) Bucket

```hcl
resource "google_storage_bucket" "assets" {
  name          = "${var.proje_adi}-assets-${var.ortam}"
  location      = "EU"
  force_destroy = var.ortam != "production"  # Production'da koruma

  versioning {
    enabled = true
  }

  lifecycle_rule {
    condition {
      age = 90  # 90 günden eski objeler
    }
    action {
      type = "Delete"
    }
  }

  cors {
    origin          = ["https://learnops.io"]
    method          = ["GET", "HEAD"]
    response_header = ["*"]
    max_age_seconds = 3600
  }

  uniform_bucket_level_access = true

  labels = local.ortak_etiketler
}

# Bucket'ı public yap (statik dosyalar için)
resource "google_storage_bucket_iam_member" "public_read" {
  count  = var.ortam == "production" ? 1 : 0
  bucket = google_storage_bucket.assets.name
  role   = "roles/storage.objectViewer"
  member = "allUsers"
}
```

### 3. Cloud SQL (PostgreSQL)

```hcl
resource "google_sql_database_instance" "main" {
  name             = "${var.proje_adi}-db-${var.ortam}"
  project          = var.proje_id
  region           = var.bölge
  database_version = "POSTGRES_15"

  settings {
    tier              = var.db_tier  # "db-f1-micro", "db-g1-small" vb.
    availability_type = var.ortam == "production" ? "REGIONAL" : "ZONAL"
    disk_size         = var.ortam == "production" ? 100 : 10

    backup_configuration {
      enabled                        = var.ortam == "production"
      start_time                     = "02:00"
      point_in_time_recovery_enabled = var.ortam == "production"
    }

    ip_configuration {
      ipv4_enabled    = false
      private_network = google_compute_network.main.id
      require_ssl     = true
    }

    database_flags {
      name  = "max_connections"
      value = "100"
    }
  }

  deletion_protection = var.ortam == "production"

  depends_on = [google_service_networking_connection.private_vpc_connection]
}

resource "google_sql_database" "learnops" {
  name     = "learnops"
  instance = google_sql_database_instance.main.name
}
```

### 4. Cloud Run

```hcl
resource "google_cloud_run_v2_service" "backend" {
  name     = "${var.proje_adi}-backend-${var.ortam}"
  location = var.bölge

  template {
    service_account = google_service_account.cloud_run_sa.email

    scaling {
      min_instance_count = var.ortam == "production" ? 1 : 0
      max_instance_count = var.ortam == "production" ? 10 : 3
    }

    containers {
      image = "${var.bölge}-docker.pkg.dev/${var.proje_id}/learnops/backend:${var.image_tag}"

      ports {
        container_port = 8080
      }

      resources {
        limits = {
          cpu    = "1"
          memory = "512Mi"
        }
      }

      env {
        name = "ENVIRONMENT"
        value = var.ortam
      }

      env {
        name = "DATABASE_URL"
        value_source {
          secret_key_ref {
            secret  = google_secret_manager_secret.db_url.secret_id
            version = "latest"
          }
        }
      }
    }

    vpc_access {
      connector = google_vpc_access_connector.main.id
      egress    = "ALL_TRAFFIC"
    }
  }

  traffic {
    type    = "TRAFFIC_TARGET_ALLOCATION_TYPE_LATEST"
    percent = 100
  }
}

# Cloud Run'ı herkese açık yap
resource "google_cloud_run_v2_service_iam_member" "public" {
  project  = var.proje_id
  location = var.bölge
  name     = google_cloud_run_v2_service.backend.name
  role     = "roles/run.invoker"
  member   = "allUsers"
}
```

## Secret Manager ile Hassas Verileri Yönetme

Şifreleri, API anahtarlarını Terraform konfigürasyonuna **doğrudan yazmayın**. Secret Manager kullanın:

```hcl
# Secret oluştur
resource "google_secret_manager_secret" "jwt_secret" {
  secret_id = "${var.proje_adi}-jwt-secret-${var.ortam}"
  project   = var.proje_id

  replication {
    auto {}
  }
}

# Secret versiyonu — gerçek değer (genellikle CI/CD ile set edilir)
resource "google_secret_manager_secret_version" "jwt_secret_v1" {
  secret      = google_secret_manager_secret.jwt_secret.id
  secret_data = var.jwt_secret  # sensitive variable

  lifecycle {
    ignore_changes = [secret_data]  # Terraform dışı değişiklikleri yoksay
  }
}

# Service account'a secret okuma yetkisi ver
resource "google_secret_manager_secret_iam_member" "cloud_run_jwt_access" {
  project   = var.proje_id
  secret_id = google_secret_manager_secret.jwt_secret.secret_id
  role      = "roles/secretmanager.secretAccessor"
  member    = "serviceAccount:${google_service_account.cloud_run_sa.email}"
}
```

## IAM: En Az Yetki Prensibi

```hcl
# Cloud Run için dedicated service account
resource "google_service_account" "cloud_run_sa" {
  account_id   = "${var.proje_adi}-cloudrun-sa"
  display_name = "LearnOps Cloud Run Service Account"
  project      = var.proje_id
}

# Cloud SQL erişimi
resource "google_project_iam_member" "cloud_run_sql" {
  project = var.proje_id
  role    = "roles/cloudsql.client"
  member  = "serviceAccount:${google_service_account.cloud_run_sa.email}"
}

# Secret Manager erişimi
resource "google_project_iam_member" "cloud_run_secrets" {
  project = var.proje_id
  role    = "roles/secretmanager.secretAccessor"
  member  = "serviceAccount:${google_service_account.cloud_run_sa.email}"
}

# Storage erişimi (assets)
resource "google_storage_bucket_iam_member" "cloud_run_storage" {
  bucket = google_storage_bucket.assets.name
  role   = "roles/storage.objectViewer"
  member = "serviceAccount:${google_service_account.cloud_run_sa.email}"
}
```

## data Source: Mevcut Kaynakları Okuma

`data` bloğu, Terraform dışında oluşturulmuş kaynakların bilgilerini okur:

```hcl
# Mevcut projenin bilgilerini oku
data "google_project" "current" {}

# Mevcut bir network'ü referans al
data "google_compute_network" "shared_vpc" {
  name    = "shared-vpc-network"
  project = "altyapi-project"
}

# Mevcut bir secret'ı oku
data "google_secret_manager_secret_version" "db_password" {
  secret = "learnops-db-password"
}

# Kullanım
resource "google_cloud_run_v2_service" "backend" {
  # ...
  template {
    containers {
      env {
        name  = "DB_PASSWORD"
        value = data.google_secret_manager_secret_version.db_password.secret_data
      }
    }
  }
}
```

## Hands-on: LearnOps Altyapısı

**Görev:** LearnOps için minimal bir GCP altyapısı tanımlayın:

1. 1 GCS bucket (assets)
2. 1 Cloud Run service (backend)
3. 1 Service account

```hcl
# Yapı şablonu — ayrıntıları doldurun:

variable "proje_id" { type = string }
variable "ortam"    { type = string; default = "development" }
variable "bölge"    { type = string; default = "europe-west3" }

locals {
  ön_ek = "learnops-${var.ortam}"
}

# 1. GCS Bucket
resource "google_storage_bucket" "assets" {
  name     = "${local.ön_ek}-assets"
  location = "EU"
  # force_destroy, versioning ekleyin
}

# 2. Service Account
resource "google_service_account" "app_sa" {
  account_id = "${local.ön_ek}-app-sa"
  # display_name ekleyin
}

# 3. Cloud Run Service
resource "google_cloud_run_v2_service" "backend" {
  name     = "${local.ön_ek}-backend"
  location = var.bölge
  # template, containers, scaling ekleyin
}

output "backend_url" {
  value = google_cloud_run_v2_service.backend.uri
}
```

## Özet

Bu bölümde şunları öğrendik:

- GCP provider kurulumu ve kimlik doğrulama yöntemleri (ADC, SA key, WIF)
- Temel GCP kaynaklarını Terraform ile yönetmeyi:
  - VPC Network ve Firewall kuralları
  - Cloud Storage bucket (lifecycle, CORS, versioning)
  - Cloud SQL PostgreSQL
  - Cloud Run service (scaling, VPC, env vars)
- Secret Manager entegrasyonu
- IAM: En az yetki prensibi ve service account yönetimi
- `data` source ile mevcut kaynakları referans almayı

Bir sonraki ve son bölümde production Terraform'un olmazsa olmazlarını — workspace, state locking, CI/CD entegrasyonu — öğreneceğiz.
