# Docker Daemon, Client, Registry

`docker run` yazdığınızda perde arkasında birden fazla bileşen devreye girer. Docker'ın mimarisini anlamak, hem komutların ne yaptığını hem de hata aldığınızda nerede bakmanız gerektiğini netleştirir. Docker üç ana bileşen üzerine kuruludur: **Daemon**, **Client** ve **Registry**.

## Docker Daemon

Docker Daemon (`dockerd`), arka planda çalışan ve tüm Docker operasyonlarını yürüten süreçtir. Container'ları oluşturmak, başlatmak, durdurmak ve silmek bu sürecin sorumluluğundadır. Image'ları yönetir, network yapısını kurar, volume'ları bağlar.

Daemon bir REST API üzerinden komut alır. Kullanıcı doğrudan Daemon'la değil; Docker Client aracılığıyla haberleşir.

Eğer Docker çalışmıyorsa ya da terminalde `Cannot connect to the Docker daemon` hatası görüyorsanız, büyük ihtimalle `dockerd` başlamamış demektir. Docker Desktop veya `systemctl start docker` ile başlatılması gerekir.

## Docker Client

Docker Client, kullanıcının Docker ile etkileşime geçtiği araçtır. Terminal üzerinden yazdığınız `docker build`, `docker run`, `docker ps` gibi komutlar bu istemci tarafından alınır ve REST API aracılığıyla Daemon'a iletilir.

```
[Kullanıcı] → docker run nginx → [Client] → REST API → [Daemon] → container başlatır
```

Client ve Daemon aynı makinede çalışabilir ya da uzak bir Daemon'a bağlanabilir. Bu ayrım Docker'ın uzak sunucularda da yönetilebilmesini sağlar.

## Docker Registry

Registry, image'lerin depolandığı ve dağıtıldığı merkezi sistemdir. En bilinen örneği **Docker Hub**'dır. Bunun yanında GitHub Container Registry (`ghcr.io`), Google Artifact Registry veya şirkete özel private registry çözümleri de kullanılabilir.

```bash
docker pull nginx        # Daemon, Docker Hub'dan nginx image'ini indirir
docker push kullanici/uygulama:1.0  # Daemon, image'i registry'ye gönderir
```

Registry mantığı ekip çalışması ve CI/CD süreçleri için kritiktir. Image yalnızca lokalde kalırsa başka biri onu kullanamaz.

## Üçü birlikte nasıl çalışır?

`docker run nginx` komutunu çalıştırdığınızda şu adımlar gerçekleşir:

1. **Client** komutu alır ve Daemon'a iletir
2. **Daemon** lokal image deposuna bakar — `nginx` var mı?
3. Yoksa **Registry**'den (Docker Hub) indirir
4. Image mevcut veya indirildi — **Daemon** container'ı başlatır
5. Çalışan container bilgisi **Client**'a döner

Bu akışı bilmek, "image neden indirilmiyor?", "container neden başlamıyor?" gibi soruları çözerken hangi katmana bakmanız gerektiğini gösterir.

## Image ve Container hatırlatması

Bu noktada iki kavramı da netleştirmek faydalıdır:

- **Image**: uygulamanın paketlenmiş şablonu. Daemon tarafından saklanır, Registry'den indirilir veya Registry'ye gönderilir.
- **Container**: image'dan üretilen çalışan örnek. Daemon tarafından yönetilir.

| Bileşen | Görevi |
|---|---|
| Docker Daemon | Container ve image operasyonlarını yürütür |
| Docker Client | Kullanıcı komutlarını Daemon'a iletir |
| Docker Registry | Image'ları depolar ve dağıtır |

## Özet

Bu bölümde Docker'ın üç temel bileşenini gördük.

- Daemon arka planda çalışır; container ve image operasyonlarını yürütür
- Client terminal komutlarını Daemon'a REST API aracılığıyla iletir
- Registry image'ların depolandığı ve paylaşıldığı sistemdir
- Bu üçü birlikte `docker run` gibi basit bir komutun arkasındaki tam akışı oluşturur
