# Docker Daemon, Client, Registry

Docker tek bir programdan değil, birbirleriyle iletişim kuran birkaç bileşenden oluşur. Bu bileşenleri anlamak, Docker'ın neden bu şekilde çalıştığını ve komutların arka planda ne yaptığını kavramayı kolaylaştırır.

## Docker Daemon

Docker Daemon (`dockerd`), arka planda çalışan asıl işleme motorudur. Image build etmek, container başlatmak, durdurmak ve yönetmek gibi tüm işlemler daemon üzerinden gerçekleşir.

Daemon doğrudan kullanıcı tarafından çağrılmaz. Bunun yerine client aracılığıyla komutlar iletilir ve daemon bu komutları uygular.

## Docker Client

Docker Client, kullanıcının Docker ile konuştuğu arabirimdir. Terminal üzerinden yazdığınız `docker run`, `docker build`, `docker ps` gibi komutlar aslında Docker Client tarafından işlenir.

Client bu komutları REST API aracılığıyla daemon'a iletir. Daemon ise isteği alır ve gerekli işlemi gerçekleştirir. Kullanıcı genellikle bu iletişimi fark etmez; ama arka planda her zaman gerçekleşir.

## Docker Registry

Registry, Docker image'larının saklandığı ve dağıtıldığı merkezi depodur. En bilinen registry Docker Hub'dır. Bunun yanı sıra GitHub Container Registry (ghcr.io) ve şirket içi özel registry çözümleri de kullanılabilir.

Registry sayesinde bir image tek kişinin bilgisayarında kalmaz; ekiple, CI sistemiyle veya başka bir sunucuyla paylaşılabilir.

## `docker run` arkada ne yapar?

Bir `docker run nginx` komutu çalıştırıldığında arka planda şu adımlar gerçekleşir:

1. Client komutu alır ve daemon'a iletir
2. Daemon lokal cache'de `nginx` image'ini arar
3. Image yoksa Docker Hub'dan indirir (pull)
4. İndirilen image'dan container oluşturur
5. Container'i başlatır

Bu akış Docker'ın istemci-sunucu mimarisini güzel özetler.

## Bileşen özeti

| Bileşen | Görevi |
|---|---|
| Docker Daemon | İşlemleri gerçekleştiren arka plan servisi |
| Docker Client | Kullanıcıdan komut alan ve daemon'a ileten arabirim |
| Docker Registry | Image'ların saklandığı merkezi depo |

## Özet

Bu bölümde Docker'ın üç temel bileşenini öğrendik.

- Daemon arka planda çalışır ve tüm container/image işlemlerini yönetir
- Client, kullanıcı komutlarını REST API üzerinden daemon'a iletir
- Registry, image'ların paylaşıldığı ve dağıtıldığı merkezi depodur
- `docker run` bu üç bileşeni bir araya getiren bir akışı tetikler
